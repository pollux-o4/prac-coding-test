버퍼 직접 접근: readline()은 시스템 버퍼에서 바로 데이터를 읽어와서 처리 과정이 단순함
메모리 효율성: input()은 매번 새로운 입력 스트림을 생성하지만, readline()은 기존 버퍼 활용
파싱 과정 생략: input()은 입력을 파싱하고 검증하는 과정이 있지만, readline()은 단순히 문자열만 읽음
시스템 호출 최소화: readline()은 시스템 레벨에서 직접 읽어와서 함수 호출 오버헤드가 적음

- input()보다 readline()이 빠른 이유 (초등학생 버전)
말하기 vs 가만히 듣기: input()은 매번 "뭐야?"라고 물어보지만, readline()은 가만히 듣기만 함
직접 꺼내기: readline()은 편지함에서 편지를 바로 꺼내지만, input()은 매번 새 편지를 기다림
메모리 아끼기: input()은 매번 새로운 상자를 만들지만, readline()은 기존 상자 재사용
검사 안 하기: input()은 "이게 맞나?" 하고 검사하지만, readline()은 그냥 받기만 함
짧은 길: readline()은 컴퓨터가 바로 가져오는 짧은 길, input()은 돌아가는 긴 길

# not 연산자와 != 연산자의 차이
- not 연산자
    - 단항 연산자 (하나의 값에만 적용)
    - 불린 반전 (True -> False)
    - Truthy/Falsy 값을 반전

- != 연산자
    - 이항 연산자 (두 값을 비교)
    - 정확히 같지 않음을 비교
    - 두 값이 다르면 True

```python
# 예시
a = 0

# not 연산자
print(not a) # not 0 -> not False -> True
print(not "") # not "" -> not False -> True
print(not "hello") # not "hello" -> not True -> False

# != 연산자
print(a != 0) # 0 != 0 -> False
print("" != "hi") # "" != "hi" -> True
print(a != 5) # 0 != 5 -> True
```

# for 반복문에서 sys.stdin
- 하나씩 꺼낼 수 있는 객체.. 를 "이터레이터"라고 함
- 파일도 이터레이터에 속함
    - ex: 텍스트 파일에서 여러 줄이 있을 때, 한 줄씩 꺼낼 수 있음
- sys.stdin은 "표준 입력"이라는 특별한 파일임

- 일반 파일 vs 표준 입력:
    - 일반 파일: 디스크에서 읽음
    - ex:
    ```python
    with open("example.txt", "r") as file:
        for line in file:
            print(line)
    ```
    
    - 표준 입력: 키보드에서 읽음
    - ex:
    ```python
    import sys
    for line in sys.stdin:
        print(line)
    ```

- sys.stdin의 동작 과정
    - 코드:
    ```python
    import sys
    for line in sys.stdin:
        print(line)
    ```
    
    - 동작:
        - 프로그램이 입력을 기다림
        - 사용자가 입력 후 enter
        - line = "hello world/n" (줄바꿈 포함)
        - print 실행
        - 다시 for 문 시작 ... 반복
        - 사용자가 Ctrl+D (EOF)
        - for문 종료

# map과 zip
- map(function, iterable1, iterabl2, ...)
- zip(iterable1, iterable2, ...)

- map: 각 요소에 함수를 적용해줌
- zip: 각 요소를 묶어줌

```python
arr = [[1,2,3], [4,5,6]]
arr2 = [[4,5,6], [7,8,9]]
result = list(map(lambda x,y: [a+b for a, b in zip(x, y)], arr, arr2))
# >>> [[5,7,9], [11,13,15]]
```

- 리스트 컴프리헨션 사용
    - [ `반복문 아래 쓰일 코드` for `할당된 변수들` in `이터러블 객체` ]
    - [ `표현식` for `변수` in `이터러블` ]
    - 예시
    ```python
    # input -> 1, 2, 3
    arr1 = []
    for _ in range(3):
        arr1.append(input())
    
    # 이걸 컴프리헨션 쓰면
    arr1 = [input() for _ in range(3)]
    ```
