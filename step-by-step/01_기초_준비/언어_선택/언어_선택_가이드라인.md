# 코딩테스트 언어 선택 가이드라인

## 🎯 언어 선택의 중요성

코딩테스트에서 언어 선택은 매우 중요합니다. 각 언어마다 장단점이 있으므로 자신의 상황에 맞는 언어를 선택해야 합니다.

## 📊 주요 언어별 비교

### 1. Python (추천 ⭐⭐⭐⭐⭐)
**장점:**
- 문법이 간단하고 직관적
- 라이브러리가 풍부 (collections, itertools 등)
- 코드 작성 속도가 빠름
- 코딩테스트에서 가장 인기 있는 언어

**단점:**
- 실행 속도가 상대적으로 느림
- 메모리 사용량이 많음

**적합한 사람:**
- 프로그래밍 초보자
- 빠른 문제 해결이 필요한 사람
- 알고리즘에 집중하고 싶은 사람

### 2. Java (추천 ⭐⭐⭐⭐)
**장점:**
- 객체지향 프로그래밍
- 강력한 타입 시스템
- 안정적이고 신뢰성 높음
- 기업에서 많이 사용

**단점:**
- 코드가 길어짐
- 초기 설정이 복잡
- 메모리 사용량이 많음

**적합한 사람:**
- Java를 이미 알고 있는 사람
- 기업 코딩테스트를 준비하는 사람
- 안정적인 코드를 원하는 사람

### 3. C++ (추천 ⭐⭐⭐⭐)
**장점:**
- 실행 속도가 매우 빠름
- 메모리 효율적
- STL 라이브러리 활용 가능
- 하드웨어 제어 가능

**단점:**
- 문법이 복잡함
- 메모리 관리가 어려움
- 디버깅이 어려움

**적합한 사람:**
- 성능이 중요한 문제를 풀어야 하는 사람
- C++을 이미 알고 있는 사람
- 시스템 프로그래밍에 관심 있는 사람

### 4. JavaScript (추천 ⭐⭐⭐)
**장점:**
- 웹 개발과 연계 가능
- 문법이 간단함
- Node.js 환경에서 실행 가능

**단점:**
- 코딩테스트에서 지원하지 않는 경우가 많음
- 성능이 상대적으로 느림

**적합한 사람:**
- 웹 개발자
- JavaScript를 이미 알고 있는 사람

## 🎯 추천 언어 선택 기준

### 초보자라면: Python
- 문법이 간단해서 알고리즘에 집중할 수 있음
- 풍부한 라이브러리로 빠른 문제 해결 가능
- 커뮤니티가 크고 자료가 많음

### 경험자라면: Java 또는 C++
- 이미 알고 있는 언어가 있다면 그 언어 사용
- 기업 코딩테스트에서 선호하는 언어
- 성능이 중요한 문제에 유리

### 특별한 목적이 있다면:
- **성능 중시**: C++
- **안정성 중시**: Java
- **개발 속도 중시**: Python

## 📚 언어별 필수 문법 정리

### Python 필수 문법
```python
# 입력 받기
n = int(input())
arr = list(map(int, input().split()))

# 리스트 컴프리헨션
result = [x for x in range(10) if x % 2 == 0]

# 딕셔너리
dict = {'key': 'value'}

# 정렬
arr.sort()  # 원본 변경
sorted_arr = sorted(arr)  # 새 리스트 반환

# 최대/최소
max_val = max(arr)
min_val = min(arr)
```

### Java 필수 문법
```java
// 입력 받기
Scanner sc = new Scanner(System.in);
int n = sc.nextInt();
int[] arr = new int[n];

// 배열 정렬
Arrays.sort(arr);

// ArrayList
ArrayList<Integer> list = new ArrayList<>();
list.add(1);
list.get(0);

// HashMap
HashMap<String, Integer> map = new HashMap<>();
map.put("key", 1);
map.get("key");
```

### C++ 필수 문법
```cpp
// 입력 받기
int n;
cin >> n;
vector<int> arr(n);

// 벡터 정렬
sort(arr.begin(), arr.end());

// 벡터
vector<int> vec;
vec.push_back(1);
vec[0];

// 맵
map<string, int> m;
m["key"] = 1;
m["key"];
```
